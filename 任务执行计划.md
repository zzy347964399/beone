## 阶段 0：环境搭建 & 数据加载（10 min）

**目标**：Git 仓库初始化，所有 26 个 CSV 正常读取，验证核心表行数。

### 步骤

1. 创建项目目录并初始化 Git
   ```bash
   mkdir mimic-iii-analysis && cd mimic-iii-analysis
   git init
   ```

2. 创建 `.gitignore`，排除大文件
   ```
   *.csv
   __pycache__/
   *.pyc
   ```

3. 安装依赖（提前准备）
   ```bash
   pip install pandas numpy plotly scikit-learn xgboost lightgbm shap
   ```

4. 写 `load_data.py`，验证核心表加载
   - ADMISSIONS、PATIENTS、ICUSTAYS 正常读取
   - LABEVENTS（76K 行）、CHARTEVENTS（758K 行）——**先只加载 header 验证 schema**，正式使用时按 itemid 过滤

### ⚠️ 关键陷阱

- CHARTEVENTS 体积极大，**禁止直接 `pd.read_csv()` 全量加载**，用 `usecols` + `chunksize` 或提前按 itemid 过滤
- PATIENTS 中 >89 岁患者的 `dob` 被 MIMIC 位移至 1800 年代（隐私保护），直接做 timedelta 减法会 int64 溢出；特征工程中需用 `dob >= 1900` 判断，位移患者用中位年龄填充并加 `age_imputed` 标志列

## 阶段 2-FE：Level 2 特征工程（20 min）

**目标**：构建两个 Track 共用的特征矩阵 DataFrame。

### 共用特征（两个 Track 均需）

**人口统计学特征**（来自 PATIENTS + ADMISSIONS）

| 特征 | 原始字段 | 处理方式 |
|------|---------|---------|
| age | dob + admittime | 正常患者（dob≥1900）算真实年龄；>89岁患者填正常患者中位年龄 |
| age_imputed | dob | 0=真实年龄，1=年龄未知(>89岁)，作为独立特征进入模型 |
| gender_M | gender | One-hot（M=1, F=0） |
| admission_type | admission_type | One-hot（EMERGENCY/ELECTIVE/URGENT） |

**Charlson 合并症指数**（来自 DIAGNOSES_ICD）

- 用 `charlson` 库或手写 ICD-9 → 疾病分类映射表计算
- 每次住院（hadm_id）聚合为一个 Charlson 分值

## 阶段 2A：Track A — 死亡率预测建模（10 min）

**目标变量**：`hospital_expire_flag`（0/1 二分类）

### 建模流程

1. **数据分割**：`train_test_split(test_size=0.2, stratify=y)`
2. **基线模型**：逻辑回归（`LogisticRegression(class_weight='balanced')`）
3. **改进模型**：XGBoost 或 RandomForest（5 min 以内跑完）
4. **评估**：AUROC、混淆矩阵、Precision/Recall/F1（重点关注死亡类别）
5. **可视化**：ROC 曲线对比图、混淆矩阵热图、特征重要性条形图

### 改进方向（时间允许则做）

- `class_weight='balanced'` 或 SMOTE 过采样处理类别不平衡
- SHAP 解释模型特征贡献（`shap.TreeExplainer`）

### 验收检查

- [ ] 特征矩阵 ≥ 24 列
- [ ] 报告逻辑回归 AUROC 并与改进模型对比
- [ ] 提供 ROC 曲线 + 混淆矩阵
- [ ] 有缺失值处理说明

## 阶段 2B：Track B — ICU LOS 预测建模（10 min）

**目标变量**：`ICUSTAYS.los`（ICU 住院天数，连续值）

### 建模流程

1. **目标变换**：`y = log1p(los)`，训练后用 `expm1` 反变换预测值
2. **基线对比**：均值预测（naive baseline）
3. **主模型**：XGBoost 或 LightGBM 梯度提升树
4. **评估**：MAE、RMSE、R²（与均值 baseline 对比）
5. **可视化**：预测值 vs. 实际值散点图、误差分布、特征重要性图

### 验收检查

- [ ] 特征矩阵 ≥ 22 列
- [ ] 报告 MAE/RMSE/R² 并与 baseline 对比
- [ ] 特征重要性可视化

## 🚨 高频陷阱速查

| 陷阱 | 说明 | 应对方法 |
|------|------|---------|
| CHARTEVENTS 爆内存 | 758K 行，直接 read_csv 可能 OOM | 先 `usecols=['itemid','icustay_id','charttime','valuenum']` 过滤，再用 `query` 筛 itemid |
| 年龄位移 | >89 岁患者 dob 被 MIMIC 位移至 1800 年代，直接做 timedelta 运算会 int64 溢出 | 用 `dob >= 1900` 区分正常/位移患者；正常患者算真实年龄，位移患者填中位年龄 + `age_imputed=1` 标志列 |
| 双系统 itemid | CareVue + MetaVision 同一指标有不同编码 | 构建 itemid 列表，`isin([211, 220045])` 统一过滤 |
| 类别不平衡 | 死亡患者远少于存活 | `class_weight='balanced'` 或 SMOTE |
| LABEVENTS 时间窗口 | 24h 窗口基于 admittime，非 charttime 绝对值 | `LABEVENTS.charttime <= admittime + timedelta(hours=24)` |
| hospital_expire_flag 空值 | 部分行为 NULL | `fillna(0)` 或 `dropna()` 后处理 |
| ICU LOS 右偏 | LOS 分布呈长尾 | 目标变量 `log1p` 变换，预测后 `expm1` 还原 |
